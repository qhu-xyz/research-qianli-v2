### Code Findings
1. Medium: Missing new source features are only warned in the loader, not failed fast. In `ml/data_loader.py:136-143`, missing columns only emit a warning; hard failure occurs later in feature prep (less local and less actionable). This is an avoidable delayed-failure path.
2. Low: Smoke data semantics for `sf_nonzero_frac` are unrealistic. In `ml/data_loader.py:51-53`, `sf_nonzero_frac` is sampled as `abs(N(0,1))`, so values can exceed `1.0` even though the feature represents a fraction. This weakens smoke-test fidelity for feature-scale assumptions.
3. Low: New failure path is not directly unit-tested. `ml/features.py:49-57` adds a critical `ValueError` branch for missing source-loader features, but there is no corresponding negative test in `ml/tests/test_features.py`. Regression risk is low but unnecessary.

### Recommendations
1. Enforce fail-fast on missing source columns in `load_data` (same place as diagnostic), so the error is immediate and closer to root cause.
2. Constrain smoke generation for `sf_nonzero_frac` to `[0,1]` (e.g., beta/uniform) and keep `is_interface` binary; this improves test realism without changing production behavior.
3. Add tests for: missing source feature columns, and end-to-end failure messaging when any of the six new features is absent.
4. Keep precision-first strategy unchanged: the strong AUC/AP/VCAP@100 profile is aligned with business objective; do not trade it for recall-heavy threshold changes.

### Gate Calibration

- `S1-VCAP@100` floor remains non-informative (`-0.0351`); tightening toward `0.0` is still reasonable once champion enforcement is active.
