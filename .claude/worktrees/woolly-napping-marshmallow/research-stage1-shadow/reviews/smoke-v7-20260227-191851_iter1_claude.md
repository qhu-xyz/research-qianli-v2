### 4. Threshold beta=0.3 — HYPOTHESIS FAILED (direction error)

The `config.json` confirms `threshold_beta: 0.3` was applied. The `meta.json` shows `threshold: 0.8203, max_fbeta: 0.9237`. The metrics are bit-for-bit identical to v0.

**Why**: At n=20 with 2 positives, the `precision_recall_curve` returns a discrete set of operating points corresponding to the 20 sample probabilities. With beta=0.3, the F-beta formula heavily weights precision: F_0.3 = 1.09 * P * R / (0.09P + R). This makes the optimizer select the highest-precision operating point, which is at threshold=0.8203 — the same as v0's beta=0.7 choice. In fact, beta=0.3 pushes MORE toward precision (higher threshold), not less.

The direction_iter1.md stated: "Reducing beta to 0.3 weights recall more heavily." This is **incorrect** per the F-beta formula. The worker's post-hoc diagnosis in changes_summary.md is accurate.

### 5. Misleading docstring in threshold.py:8

The docstring says `beta: float = 0.7` with the comment `"0.7 = moderate recall/precision balance"`. This is misleading — beta=0.7 moderately favors **precision** over recall. This confusion likely contributed to the flipped hypothesis. **LOW severity but should be corrected** to prevent future misdirection.

## Recommendations for Iteration 2

### CRITICAL: Fix the beta direction

Use `threshold_beta > 1` to favor recall over precision. Recommended starting point: **beta=2.0**.

Rationale:
- F_2.0 = 5 * P * R / (4P + R) — heavily weights recall.
- At n=20 with binding_rate=0.1, we need the optimizer to accept low precision in exchange for non-zero recall.
- beta=3.0 is also worth considering if beta=2.0 doesn't lower the threshold enough.

### IMPORTANT: Monitor S1-BRIER carefully

Any threshold that produces positive predictions will change the Brier score. With only 0.02 headroom, this gate is very likely to flip when recall improves. If it does, the orchestrator should:
1. Document the Brier value at the new threshold.
2. Assess whether the floor is too tight for SMOKE_TEST (n=20).
3. Recommend floor recalibration at HUMAN_SYNC if the Brier increase is small and proportionate to the recall gain.
