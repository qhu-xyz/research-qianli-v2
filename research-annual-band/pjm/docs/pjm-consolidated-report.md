## Pxx Definition

**Pxx = single bid price with xx% clearing probability on training data.**
Each level is one bid price, NOT a pair.

## Frozen Spec

### Baseline

| Round | Formula | Notes |
|-------|---------|-------|
| R1 | `w(flow,bin) × mtm_1st_period + (1-w) × recent_1` | Blend with March DA revenue |
| R2-R4 | `mtm_1st_period` (pure) | No blend needed |

**R1 `recent_1`:** March of year N DA congestion (sink − source) × 12 (annual scale).
The most recent single month before the June auction. Scaled ×12 to match annual baseline.

**Why `recent_1` for R1:** The R1 baseline (`mtm_1st_period` = LT yr1 R5 MCP from ~1 year prior)
is stale. March DA congestion captures current congestion conditions that the year-old baseline
misses. Recency gradient: March alone (×12) > Jan-Mar (×4) > Oct-Mar (×2) > trailing 12mo.

**Why NOT blend for R2-R4:** 1(rev) oracle w = 1.00 for all R2-R4 cells. Prior-round MCP
already incorporates DA revenue information (same finding as MISO R2/R3).

### Calibration

**(bin, flow_type, class_type)** for all rounds, all bins.

Unlike MISO (where class was noise), PJM's 3 class types (onpeak, dailyoffpeak, wkndonpeak)
have structurally different residual distributions — dropping class worsens P10-P50 by 3-4pp.

Fallback: if (bin, flow, class) cell < 200 rows, pool to (bin, flow).

### Other Parameters

- 5 quantile bins on training |baseline|
- Expanding window, min 2 training PYs (first evaluable PY = 2019)
- Governance cap: ±50,000 annual $ on bid prices
- Scale: all annual $ (native for PJM period type `a`)

## Known Limitations

### Structural failures (q5)

**R1 PY2024 prevail q5:** P50=34.2%, bias=+1,081. Massive under-prediction — MCP far exceeded
baseline. Single worst holdout cell.

**R1 PY2025 counter q5 onpeak:** P50=57.6%, bias=-3,485. Extreme negative bias — MCP much
lower than baseline.

**R2-R4 PY2025 prevail q5:** P95 ranges 85-89% (target 95%). Bands too narrow for this year.

### PY2022 (dev) is catastrophic

R1 PY2022 prevail q5: P95=69.0% (target 95%). The LT yr1 R5 baseline massively under-predicted
PY2022 MCPs. This is a structural outlier that poisons the expanding window for PY2023-2025.

### Class-specific patterns

PJM's 3 class types have different failure modes for counter q5:
- onpeak: extreme negative bias (PY2025: -3,485)
- wkndonpeak: moderate over-coverage (PY2023: P50=80%, bias=-555)
- dailyoffpeak: mixed

## What We Tried

### 1(rev) trailing 12 months — REJECTED for R1
- `corr(residual, 1(rev))` = -0.10 overall — doesn't predict the error
- Per-cell correlation is positive (+0.2-0.4) but Simpson's paradox makes overall near-zero
- `corr(baseline, 1(rev))` = +0.73 — both track the same signal, no independent information
- Oracle w for R2-R4 = 1.00 (useless, same as MISO)

### Revenue window search — found `recent_1`
Tested 6 windows: trail_12, recent_6, recent_3, recent_1, winter, summer, prior_py.

| Window | Months | Scale | q4+q5 P50 |miss| |
|--------|--------|-------|:---:|
| pure (no blend) | — | — | 13.1pp |
| trail_12 | Apr N-1 to Mar N | ×1 | 13.0pp |
| recent_6 | Oct N-1 to Mar N | ×2 | 13.0pp |
| recent_3 | Jan-Mar N | ×4 | 10.0pp |
| **recent_1** | **Mar N only** | **×12** | **9.4pp** |

Clear recency gradient: fresher data = better predictor. March alone is noisier (1 month)
but captures the current regime most accurately.

### Calibration cells — A wins for PJM
Unlike MISO where pooling flow helped q4-q5:
- A: (bin,flow,class) → P50 |miss| = 13.1pp (with blend_r1: 9.4pp)
- D: (bin) → P50 |miss| = 16.9pp
- PJM needs the flow+class split because 3 class types have structurally different distributions
