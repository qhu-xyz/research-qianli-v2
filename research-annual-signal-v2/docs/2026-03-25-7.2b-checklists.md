Cutoff contract to preserve exactly:
- `get_history_cutoff_date()` returns the last included day.
- The daily loader expects an exclusive stop boundary via `d >= cutoff_date: break`.
- So production must pass `last_included + timedelta(days=1)` into the daily loader.
- Example for `2025-06 / R1`:
  - close date = `2025-04-08`
  - last included day = `2025-04-07`
  - daily loader cutoff argument = `2025-04-08`
  - loaded dates = `2025-04-01` through `2025-04-07`
  - excluded dates = `2025-04-08+`
- This is correct and should match research exactly. Do not "fix" it away.

## 2.11 Determinism contract (UPDATED 2026-03-25)

The research runner now pins seeds. For byte-exact reproduction, match these:

- LightGBM base: `seed=42, deterministic=True`
- LightGBM specialist: `seed=42, deterministic=True`
- No `np.random.seed` needed (no numpy randomness in the pipeline)
- `num_threads=4` with `deterministic=True` is reproducible on same hardware
- Training data concatenation order: PYs ascending, quarters `aq1→aq4` within each PY
- LambdaRank groups: `group_by(["py", "aq_label"], maintain_order=True)`
- Deviation feature anchor tie-break: `group_by("branch_name").first()` after `sort(descending=True)`
- External signal sort: `sort(["tier", "rank"], descending=[False, auto_detected])` — auto-detect from tier-rank correlation

Reference artifacts (with seeds):
- `/opt/tmp/qianli/miso/trash/annual_candidate_7.2b/results.parquet`
- `/opt/tmp/qianli/miso/trash/annual_candidate_7.2b/branch_level.parquet`

Cross-hardware note: `deterministic=True` guarantees reproducibility on the same machine. Across different CPU architectures, floating-point summation order may differ. If cross-hardware exact match is needed, use `num_threads=1` (4x slower).
