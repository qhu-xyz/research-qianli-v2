# 7.2b Publication Checklist And Manifest Draft

**Date**: 2026-03-25  
**Purpose**: publication checklist, manifest draft, and operating procedure for `7.2b`  
**Status**: review draft

## 1. Implementation Checklist

### 1.1 Branch-level scorer layer

- Add a publish-facing adapter over the existing `7.2b` branch scorer.
- Train base model on historical cells.
- Train specialist model on historical true-dormant cells.
- Score the target publish cell ex ante.
- Return:
  - `branch_name`
  - `base_score`
  - `specialist_score`
  - `is_true_dormant`
  - `origin`
  - `shadow_price_da`
  - `da_rank_value`

### 1.2 Provenance and expansion

- Assign branch provenance before expansion.
- Expand branches to constraints.
- Carry provenance onto every expanded constraint.
- Carry both branch scores onto every expanded constraint.

### 1.3 Constraint-tier assembly

- Build `base_pool`.
- Build `specialist_pool`.
- Sort both pools deterministically.
- Fill tier 0 with:
  - `50` specialist-origin constraints
  - `150` base-origin constraints
- Repeat for tiers 1 through 4.
- Backfill specialist shortfall from base within the same tier.
- Lock each tier before moving on.
- Within each tier:
  - specialist-origin block first
  - base-origin block second
- Specialist pool tie-break:
  - `specialist_score` desc
  - `base_score` desc
  - `constraint_id` asc
- Note:
  - sibling constraints from the same branch share the same branch scores
  - `constraint_id` is therefore the within-branch ordering key

### 1.4 Online checks during walk

- Enforce branch cap during selection.
- Enforce grouped SF dedup during selection.
- Do not prefilter pools and then fill tiers.
- Remove selected constraints from remaining pools.
- Share branch-cap and SF-dedup state across:
  - the specialist sub-walk
  - the base sub-walk
  within the same tier

### 1.5 Publisher wiring

- Add a separate `publish_signal_72b()`.
- Preserve existing `7.1b` behavior.
- Use existing metadata and schema machinery where compatible.
- Replace the single-stream `v0c` ranking path for `7.2b`.

### 1.6 Release/test wiring

- Add `releases/miso/annual/7.2b/manifest.json`.
- Add `releases/miso/annual/7.2b/smoke_test.json`.
- Add unit tests for the tier walk.
- Add one integration test for a full publish cell.

## 2. Publication Checklist

### 2.1 Branch-level preconditions

- Base model training completed.
- Specialist training completed.
- No skipped training cells in publish mode.
- No deviation-feature zero-fill fallback in publish mode.
- Branch scorer outputs are deterministic.
- Publish mode hard-fails on missing required deviation features or training inputs.

### 2.2 Input data preconditions

- collapsed universe exists
- CID mapping exists
- constraint limits exist
- density score exists
- SF exists
- required round partitions exist

### 2.3 Tier composition checks

- Exactly `5` tiers.
- Exactly `200` constraints per tier unless pool exhaustion is explicit.
- Target `150 base-origin + 50 specialist-origin` per tier when supply allows.
- Specialist shortfall is backfilled from base only within the affected tier.
- If pool exhaustion occurs, published count below `1000` is allowed only if the
  shortfall is recorded explicitly in the audit.

### 2.4 Online-selection checks

- Branch cap enforced during walk.
- Grouped SF dedup enforced during walk.
- No later tier reshuffles earlier tier.
- Deterministic ordering reproducible across reruns.

### 2.5 Output checks

- constraints parquet passes schema
- SF parquet passes schema
- `constraint_limit` is non-null
- every published constraint has SF coverage
- final rank/tier columns reflect `7.2b` policy, not `v0c_score`

### 2.6 Supported-grid checks

- Supported `(planning_year, aq_quarter, class_type, market_round)` cells are explicit.
- Any exclusions are explicit.
- Smoke test covers the intended `7.2b` surface.

### 2.7 Audit outputs

- Per-tier origin counts
- Branch-cap rejection counts
- SF-dedup rejection counts
- Specialist shortfall/backfill counts
- Final published constraint count

## 4. Operating Procedure

### Step 1: Build branch-level scores

- train base and specialist from historical cells
- score the target publish cell
- assign branch provenance

### Step 2: Expand to constraints

- join CID mapping
- join limits
- inherit branch scores and origin

### Step 3: Join publish metadata

- flow direction
- shadow sign
- bus key
- bus_key_group
- density score
- SF

### Step 4: Build publication pools

- build `base_pool`
- build `specialist_pool`
- sort each pool deterministically

### Step 5: Fill tiers sequentially

- fill tier 0
- fill tier 1 from remaining constraints
- continue through tier 4

Rules:

- branch cap checked online
- grouped SF dedup checked online
- branch-cap and SF-dedup state shared across specialist and base sub-walks
- specialist underfill backfilled from base within the same tier
- specialist-origin block emitted before base-origin block within each tier

### Step 6: Validate outputs

- schema validation
- SF coverage validation
- non-null validation
- per-tier audit validation

### Step 7: Write outputs

- constraints parquet
- SF parquet
- smoke-test result
- publication audit report

## 5. Confirmed Review Decisions

1. Use a separate `publish_signal_72b()`.
2. Within-tier order is:
   - specialist-origin block first
   - base-origin block second
3. Pool exhaustion may stop publication below `1000`; record the shortfall explicitly.
4. The manifest must record that the constraint-tier assembly is production-only and was not evaluated in research.
