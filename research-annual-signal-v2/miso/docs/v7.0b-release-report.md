## 1. What Changed

V7.0B adds **supplement key matching** to the DA CID → branch mapping layer. This is the only change.

### The problem V7.0B solves

When evaluating the signal against realized DA shadow prices, some DA constraints have new constraint_ids (CIDs) not in the SPICE bridge. In V7.0, these DA CIDs were classified as "unmapped" — their binding value was invisible to training and evaluation.

Many of these DA CIDs monitor branches that already exist in the SPICE universe under different CIDs. Using `MisoDaShadowPriceSupplement` structured keys, we can construct the branch name and match it to a known SPICE branch.

### How it works

```
DA CID (not in bridge)
  → Load supplement keys from MISO_DA_SHADOW_PRICE_SUPPLEMENT.parquet
  → XF (transformer): branch = key1 + key3
  → LN (line):        branch = key2 + key3
  → Normalize whitespace, match against SPICE bridge branches
  → If matched: credit DA SP to that branch
```

### What this affects

The supplement matching changes the **DA → branch** path used by GT and history features:
- `realized_shadow_price` increases for branches with recovered DA SP
- `shadow_price_da` and `da_rank_value` change (more historical DA credited)
- `bf_12` / `bfo_12` may change (more months counted as binding)
- v0c scores change (da_rank_value is 40% of v0c)
- Different branch ranking → different SPICE CIDs published

### What this does NOT affect

- The **branch → SPICE CID** publication path is unchanged
- Published CIDs are always SPICE CIDs from the density universe
- v0c formula weights (0.40/0.30/0.30) unchanged
- Density features, limits, NB detection unchanged
- The density universe (`load_collapsed`) unchanged

The holdout year (2025-06) has the most training data affected because it accumulates the most historical months with supplement-recovered DA.

### Holdout (3 groups, PY 2025-06)

| Metric | V7.0 | V7.0B | Delta |
|--------|:---:|:---:|:---:|
| **Onpeak** | | | |
| VC@400 | 0.7535 | 0.7456 | -0.008 |

| Recall@400 | 0.5395 | 0.5211 | -0.018 |
| **Offpeak** | | | |

| Abs_SP@400 | 0.6885 | 0.7158 | **+0.027** |
| Recall@400 | 0.5890 | 0.5710 | -0.018 |

### Interpretation

- **Abs_SP@400 improves +2.6-2.7pp on holdout**: More DA SP is now credited to modeled branches, so the honest coverage metric gets better. This is the primary intended effect.
- **VC@400 drops -0.6 to -0.8pp on holdout**: The ranking reshuffles. Some branches that were previously ranked high get displaced by branches that gained recovered DA SP. VC measures capture within the branch universe — it can move in either direction when the ranking changes.
- **Dev changes are negligible**: Earlier PYs have very few recovered DA CIDs, so training labels and features barely change.

## 5. Coverage Impact

The supplement matching reduces the "unmapped DA CID" gap:

| PY | CID-unmapped SP% | After supplement recovery | Residual |
|----|:---:|:---:|:---:|
| 2019-2024 | 0.1-2.7% | 0-1.6% recovered | 0.1-2.0% residual |
| **2025-06** | **29.2%** | **22.4% recovered** | **6.7% residual** |

The 6.7% residual represents DA constraints on branches that genuinely don't exist in any SPICE planning model. This is the irreducible gap.
