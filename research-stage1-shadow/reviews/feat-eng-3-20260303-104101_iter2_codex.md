### Code Findings
1. [Medium] Temporal leakage fallback remains possible when `auction_month` is absent: row-proportion split (`fit_df = train_data[:split]`, `val_df = train_data[split:]`) can mix chronological regimes and invalidate train/val separation assumptions. File: `ml/data_loader.py:191-195`.
2. [Medium] Interaction computation is all-or-nothing: if any interaction feature is requested, all interaction formulas are executed, which introduces unnecessary hard dependencies on columns not required by the active feature subset and can raise avoidable column errors. File: `ml/features.py:42-56`.
3. [Low] New interaction feature tests validate presence/count but not formula correctness against source columns, so regressions in computation logic could pass CI undetected. Files: `ml/tests/test_config.py:4-56`, `ml/tests/test_data_loader.py:24-32`, `ml/tests/test_features.py:9-30`.

### Recommendations
1. Promote v0009 over v0008 for f0/onpeak under current gate policy; Group A ranking quality is improved and tail-safe.
2. Add a hard assertion for chronological split in real mode (fail fast if `auction_month` missing) instead of fallback row split.
3. Refactor interaction creation to compute only interactions actually present in `config.features` (feature-wise conditional expressions).
4. Add targeted unit tests for derived interaction formulas (`band_severity`, `sf_exceed_interaction`, `hist_seasonal_band`) using controlled numeric fixtures.
5. Continue monitoring the same weak months (2021-04, 2022-09, 2022-12); current gains are real but not yet structurally uniform.
