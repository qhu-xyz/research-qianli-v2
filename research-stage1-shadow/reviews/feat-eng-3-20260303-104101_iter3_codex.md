### Code Findings
1. **High: Three-layer gate logic can pass with missing eval months (coverage hole).**
   - `check_gates_multi_month` evaluates only months present in `per_month` and does not enforce full primary-month coverage (`ml/compare.py:170-181`, `ml/compare.py:233-244`).
   - `run_benchmark` tracks skipped months but `run_comparison` does not fail or penalize missing months (`ml/benchmark.py:179-186`, `ml/benchmark.py:213-215`, `ml/compare.py:529-538`).
   - Impact: a version can avoid hard months and still pass Group A.
2. **Medium: Benchmark path ignores `threshold_scaling_factor`, diverging from pipeline semantics.**
   - Benchmark thresholding calls `find_optimal_threshold(..., beta=threshold_beta)` without scaling factor (`ml/benchmark.py:75-77`).
   - Pipeline path includes scaling factor (`ml/pipeline.py:105-110`).
   - Impact: threshold-dependent metrics (REC/CAP) are not reproducible between single-run and benchmark when scaling override is used.
3. **Medium (statistical): threshold is tuned and evaluated on the same validation split.**
   - Benchmark tunes threshold then evaluates on identical `val_df` (`ml/benchmark.py:75-82`); pipeline does the same (`ml/pipeline.py:104-125`).
   - Impact: optimistic bias for threshold-dependent metrics (Group B CAP/REC), which already show instability.
4. **Low: Missing-value handling is null-only, not NaN-aware.**
   - Feature prep uses `.fill_null(0)` but does not normalize NaN (`ml/features.py:79`).
   - Impact: potential silent NaN propagation from source loader, especially in real-mode edge months.

### Recommendations
1. Keep **v0009** as champion; treat v0010 as a null-result confirmation run (no meaningful Group A uplift).
2. Enforce strict month coverage before gate checks: require `n_months == n_months_requested == len(primary_eval_months)` or auto-fail Group A.
3. Thread `threshold_scaling_factor` through `run_benchmark` and `_eval_single_month` for consistency with `run_pipeline`.
4. For statistical rigor, separate threshold-tuning and final scoring windows (or at minimum label Group B metrics as optimistic due to reuse of validation data).
5. Continue targeted monitoring of 2022-12 regime weakness; it remains the dominant tail-risk month for AUC/VCAP/NDCG.

### Gate Calibration
- **Noise tolerance (0.02):** acceptable but loose for current maturity; observed inter-version Group A bot2 moves are mostly ~0.001–0.004. Consider tightening to **0.015** to improve sensitivity without overreacting.
- **Group A mean floors:** no immediate change needed (all have healthy mean headroom).
- **S1-VCAP@100 mean floor:** recommendation to tighten toward `0.0` remains reasonable given sustained positive levels.
- **S1-CAP@100 / S1-CAP@500 (Group B):** persistent structural fails support relaxing these monitor floors (as prior logs suggest) or explicitly de-emphasizing them relative to ranking metrics aligned to precision-first objectives.
