### Recommendations
1. Fix gated-mode semantics to match design: gate regressor training on classifier-predicted positives (preferably out-of-fold predictions) rather than true bind labels.
2. Correct `R-REC@500` to use regressor-only ranking (`pred_shadow_price`) so monitor metrics are interpretable.
3. Add explicit anti-leakage assertions in data split logic (no overlap among fit/val/test months after loader boundaries are resolved).
4. Add regressor feature-importance emission to pipeline outputs so feature engineering decisions can be justified quantitatively.
5. Do not treat v0009 as novel evidence relative to v0008; compare against v0007 is valid for promotion, but this is operationally a duplicate registration.
6. Unified regressor mode is worth a controlled ablation only after fixing items 1-3; otherwise comparisons are confounded by metric-definition and split risks.

### Gate Calibration
Current `noise_tolerance=0.02` is not scale-aware. With current magnitudes:
- EV-VC@100 bottom_2 champion is 0.0071, so 0.02 makes L3 effectively non-binding.
- C-RMSE/C-MAE bottom_2 are ~5k/~2k, so 0.02 is negligible.

Recommendation: switch L3 tolerance to metric-scale-aware rule (e.g., `max(abs_floor, pct * |champ_bottom_2_mean|)`), with per-metric caps. This keeps L3 meaningful for both small-scale ratio metrics and large-scale error metrics.
