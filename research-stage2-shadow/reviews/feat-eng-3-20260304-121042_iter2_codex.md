### Recommendations
1. Promote v0012 over v0011 on current gate policy: all Group A layers pass, EV-VC@500 binding risk is materially reduced, and calibration/Spearman did not regress.
2. Next iteration should fix gated-training semantics: gate regressor fit set using classifier predictions on train data (not true labels), then re-benchmark for unbiased train/inference alignment.
3. Add per-version feature importance output for regression reviewability (currently unavailable for v0012), and require it in the promotion checklist.
4. If EV-VC@100 recovery is required, tune within RegressorConfig only (e.g., small depth/min_child_weight or colsample sweeps) while preserving the EV-VC@500 tail gains.
5. Unified regressor mode is worth a controlled ablation after item 2, since current gated semantics are label-dependent and may be masking regime behavior.

### Gate Calibration
Do not recalibrate floors downward to v0011/v0012 means. Current evidence still supports the existing floors, and downward recalibration would ratchet standards down for EV-VC@500/EV-NDCG/Spearman.

`noise_tolerance=0.02` remains non-scale-aware versus observed variance and metric scales: it is overly permissive for small-scale EV tails (especially EV-VC@100 bottom_2 ~0.01) and effectively negligible for large-scale loss metrics (C-RMSE/C-MAE). Use a hybrid tolerance per metric, e.g. `max(abs_floor, pct * |champ_bottom_2|)`, with explicit caps by metric family.
