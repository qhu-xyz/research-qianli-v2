### Code Findings
1. **High: regressor training gate uses true labels (selection leakage / train-inference mismatch).** In gated mode, phase 4 selects training rows using `binding_mask = y_train_binary == 1` (`ml/pipeline.py:195-209`). That uses ground-truth binding labels, but inference gates on classifier predictions (`test_proba >= threshold`, `ml/pipeline.py:249-253`). This mismatch can bias regressor fitting and does not match the stated "train on classifier-predicted positives" design.
2. **Medium: feature importance pipeline is wired but never populated.** `ml/benchmark.py` expects `_feature_importance` in month metrics (`ml/benchmark.py:139-142`) and writes `feature_importance.json` only if present (`ml/benchmark.py:177-200`), but no code path emits this key (`ml/evaluate.py`, `ml/pipeline.py`). Result: no artifact to validate whether regressor-specific features are contributing.
3. **Medium: frozen-classifier guardrails are weak in code/tests.** Classifier remained unchanged this iteration (good), but enforcement is brittle: `ClassifierConfig` is mutable dataclass in code (`ml/config.py:111-179`), while tests assume frozen + 13-feature config (`ml/tests/test_config.py:18-65`). This mismatch means tests are stale and cannot reliably enforce the freeze constraint.
4. **Low: config override docstring overstates classifier support.** `_apply_config_overrides` says classifier overrides are supported (`ml/pipeline.py:333-334`), but implementation only applies top-level and regressor keys (`ml/pipeline.py:337-350`).

- `C-MAE` improved in 9/12 months, worsened in 3/12; worst month 2022-06 (2293.16, above tail ceiling 2283.26).
- `Spearman` improved only in 3/12 months and declined in 9/12 months; weakest month is 2021-11 (0.2635).

Feature importance: unavailable for v0005 (no `registry/v0005/feature_importance.json`), so contribution of regressor-only features cannot be validated from artifacts.

### Recommendations
1. Fix gated regressor training to use classifier-predicted positives (preferably out-of-fold predictions on training months) rather than true binding labels.
2. Keep the v0005 L2 default change (`reg_lambda=5`, `min_child_weight=25`) as the current best regressor-only direction, but run a narrow sweep to recover `Spearman` margin (e.g., `reg_lambda` 4-6, `min_child_weight` 20-30).
3. Add explicit regressor feature-importance export from the trained model (gain/cover) into per-month metrics so ranking-quality changes can be attributed.
4. Add a freeze-invariant test that hashes/compares `ClassifierConfig` between champion and candidate config at benchmark time.
5. After leakage fix, run one controlled `unified_regressor=true` experiment as architecture exploration; current blocker is rank-correlation stability, not top-K value capture.

### Gate Calibration
- Current `noise_tolerance=0.02` is not scale-aware:
  - For high-is-better metrics, it is very lenient on bottom-2 tails (e.g., `EV-VC@100` champion bottom-2 is 0.0068, so allowed regression floor becomes negative).
  - For lower-is-better metrics (`C-RMSE`, `C-MAE`), 0.02 is effectively zero at the observed scale (~1000-6000), causing L3 fails despite better means.
- Use metric-scaled L3 tolerance (for example: max of absolute floor and relative tolerance by metric variance), and separately verify `registry/gates.json` provenance against calibration notes in `memory/hot/gate_calibration.md` before promotion decisions.
