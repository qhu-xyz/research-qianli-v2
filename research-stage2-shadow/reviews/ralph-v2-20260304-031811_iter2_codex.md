### Recommendations
1. Resolve provenance mismatch before any further gate interpretation: re-run and register a clean v0006 (or v0007) with explicit `config.json` verification that `reg_alpha=1.0` if that is the intended hypothesis.
2. Keep classifier frozen as-is; continue iterating only `RegressorConfig` (compliant with constraint).
3. Prioritize Spearman recovery with minimal EV-VC damage: test `max_depth=4` or `min_child_weight`/`reg_lambda` relaxation around current setting, but require full 12-month verification (screen-only winner selection was not predictive here).
4. Add artifact-level integrity checks in benchmarking: fail run if `changes_summary` claimed overrides do not match saved `registry/<version>/config.json`.
5. Implement and persist regressor feature importance (or SHAP summary) per month so regressor-specific feature contribution can be reviewed quantitatively.
6. Given regularization-axis stagnation, `unified_regressor=true` is a reasonable next exploration branch (monitor C-RMSE/C-MAE tails and EV-VC@100 impact), but keep Group A as promotion driver.

### Gate Calibration
Current calibration remains statistically brittle:
- Absolute `noise_tolerance=0.02` is scale-appropriate for Spearman/EV metrics but not for C-RMSE/C-MAE magnitudes, where it effectively enforces near-exact tail matching.
- Suggested adjustment (HUMAN_SYNC): retain existing Group A floors for this batch governance, but migrate L3 to metric-scaled tolerance (e.g., percentage of champion bottom_2_mean, with per-metric caps) for the next calibration cycle.
- Spearman specifically is currently a razor-edge blocker (miss 0.0008); if governance intent is “non-regression within noise,” flooring exactly at champion mean should be replaced with a variance-aware margin.
