# Claude Review — Iteration 3 (batch ralph-v2-20260304-031811, v0007)

## Summary

v0007 is the **first promotable version** in this batch, passing all 4 Group A gates on all 3 layers. The iteration successfully decomposed the v0005 parameter bundle (reg_lambda=5.0 + mcw=25) and identified that **reg_lambda was the Spearman-compressing parameter, not mcw**. By reverting reg_lambda to v0's default of 1.0 while retaining mcw=25, v0007 recovers Spearman above the gate floor (+0.0004) while preserving meaningful improvements on EV-VC@500 (+6.2%), EV-NDCG (+0.5%), and a slim EV-VC@100 gain (+1.3%).

However, the improvements are **extremely marginal on Spearman** (0.3932 vs 0.3928 floor, only +0.0004) and the version trades away most of v0005's EV-VC@100 advantage (0.0699 vs 0.0735, -4.9%). The promotion is technically valid but the version barely clears the gates. The real story is that mcw=25 provides EV-VC@500/EV-NDCG benefit independent of L2, and reg_lambda=1.0 (v0's default) is needed to preserve Spearman. The combination is a modest but defensible improvement over v0.

## Gate Calibration

### Current State: Still Dysfunctional but v0007 Passes

The gates remain set at v0's exact mean, creating near-zero headroom. v0007 passes Spearman by only +0.0004 (0.01%). This is a single random-seed's worth of variance — a re-run might fail.

### Specific Concerns

1. **Spearman floor = 0.3928**: v0007 clears by 0.0004. A version with genuinely better EV-VC but Spearman 0.3925 would be blocked. The floor should be ~0.342 (0.87x v0 mean) as recommended in gate_calibration.md.

2. **EV-VC@100 floor = 0.0690**: v0007 clears by 0.0009. Again, nearly no headroom. v0 itself fails this gate on floating-point precision.

3. **Tail floors are more reasonable**: EV-VC@100 tail_floor=0.000114, Spearman tail_floor=0.2649. These are set at or near v0's per-month minimums, which is a reasonable safety level.

4. **L3 noise_tolerance=0.02 is not scale-aware**: As noted in gate_calibration.md, 0.02 is generous for EV-VC@100 (bot-2 ~0.007) but meaningless for C-RMSE (bot-2 ~5300). This should be percentage-based (e.g., 5% of champion's bot-2).

### Recommendation
Recalibrate Group A floors to ~0.87-0.90x v0 mean at HUMAN_SYNC. The current gates have now blocked 2 iterations (v0005, v0006) and v0007 passes by a margin indistinguishable from noise. The gate system should identify genuinely bad versions, not penalize normal variation.

2. **Recalibrate gates (HUMAN_SYNC required)** — the current gates are at v0 exact mean, making promotion nearly impossible for any version with normal metric variance. Fix this before the next batch.
