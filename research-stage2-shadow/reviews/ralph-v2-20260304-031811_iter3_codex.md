### Recommendations
1. Keep `v0007` as the best current candidate under existing gates, but treat it as **narrow-pass** on Spearman and tail safety.
2. Add an explicit no-leak assertion/report in data loading: prove that training/validation auction months are strictly before the target test month for each eval month.
3. Wire regressor feature-importance output into per-month metrics/artifacts so feature contribution can be reviewed each iteration.
4. Either wire `value_weighted` fully or remove it from active search space; currently it can create false-positive experiment narratives.
5. Next search should prioritize tail robustness on months 2021-11 and 2022-06 (not further mean chasing), since Group A tail margins are already at limit.
6. Unified regressor mode should not be primary next move without fresh evidence; current artifacts provide no positive signal for it under this gate regime.

### Gate Calibration
Current calibration still has scale/pathology issues:
- `noise_tolerance=0.02` is too loose for tiny-scale metrics in L3 (EV-VC@100 threshold becomes negative: `0.0068 - 0.02 = -0.0132`).
- The same `0.02` is too strict for large-scale lower-is-better metrics (C-RMSE/C-MAE), effectively requiring near-identical worst-month errors.

Suggested calibration adjustment:
- Use metric-scaled L3 tolerance (for example `max(absolute_min, relative_pct * |champ_bottom2|)`) with separate defaults for bounded [0,1] metrics vs error-scale metrics.
- Keep current tail safety (`tail_max_failures=1`) for Group A; it is the main useful robustness control in this batch.
