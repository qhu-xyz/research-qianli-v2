## Summary

v0005 adds three interaction features (`overload_x_hist`, `prob110_x_recent_hist`, `tail_x_hist`) to the existing 34-feature set. Hypothesis A (add only) beat Hypothesis B (add + prune) during the 2-month screening phase, primarily driven by better weak-month performance on 2022-06 (2.3x more value capture). The full 12-month benchmark shows **uniform small improvements across all metrics** with no regressions -- a clean, low-risk iteration.

However, the improvement is insufficient to cross the Tier-VC@100 floor. The mean increased from 0.0708 to 0.0746 (+5.4%), but the floor is 0.0750 -- a shortfall of 0.0004. This is tantalizingly close but still a Layer 1 failure. The tail metrics improved substantially (bottom_2_mean +64%), indicating the interaction features help most on difficult months. The core limitation is that feature engineering alone cannot overcome what appears to be a class weight / model capacity bottleneck for tier 0/1 discrimination.

## Tier Classification Quality

### Tier-Recall@1: Catastrophically Low (0.045)

Tier-Recall@1 is 0.045 -- worse than v0's 0.047. The model is missing ~95% of tier 1 (strongly binding, $1000-3000) constraints. This is the single biggest quality gap in the pipeline.

**Root cause:** With class weights {0:10, 1:5, 2:2, 3:1, 4:0.5}, the tier 1 weight (5) is likely insufficient given its class frequency. Tier 1 constraints sit in a difficult boundary between moderate (tier 2) and heavy (tier 0) binding -- the model defaults to predicting adjacent tiers. This is a class weight / decision boundary problem that **cannot be solved by FE alone**.

### Tier-Recall@0: Adequate (0.440)

Tier-Recall@0 improved marginally from 0.439 to 0.440. The model captures ~44% of heavily binding constraints, which is acceptable but leaves room for improvement. The high class weight (10) is doing its job.

### QWK: Stable (0.371)

Ordinal consistency is stable. The interaction features don't change the ordinal structure, which is expected -- they amplify existing signals rather than introducing new ordinal information.

## EV Ranking Quality

- **Tier-VC@100 = 0.0746**: The top-100 value capture improved 5.4% but remains below the 0.075 floor. The ranking is still not capturing enough high-value constraints in the top positions.
- **Tier-VC@500 = 0.2329**: Passing comfortably. The broader ranking is adequate.
- **Tier-NDCG = 0.7751**: Good ranking quality overall. The 0.5% improvement shows the interaction features marginally help position-weighted ranking.

## Feature Importance Analysis

The 3 new interaction features are products of already-dominant features:
- `overload_x_hist = expected_overload * hist_da` -- both parents are in top-15 by importance
- `prob110_x_recent_hist = prob_exceed_110 * recent_hist_da` -- recent_hist_da is #1 by importance (21.1%)
- `tail_x_hist = tail_concentration * hist_da` -- hist_da is #2 (13.3%)

The small but consistent improvement (+5.4% VC@100) suggests XGBoost was already approximating these interactions via multi-level splits but benefits from having them pre-computed. The effect is real but modest -- consistent with the "low risk, low ceiling" prediction in the direction document.

**Concern:** The direction notes a WARNING that "the 3 new features are computed by `compute_interaction_features()` but filled with 0 initially from missing raw data columns." This was flagged in MEMORY.md. If the source columns (`expected_overload`, `prob_exceed_110`, `tail_concentration`, `hist_da`, `recent_hist_da`) are sometimes missing from the data loader, the interaction features would be zero-filled, rendering them useless for those samples. The 5.4% improvement suggests they ARE populated in at least some training months, but this should be verified.

## Statistical Rigor

With 12 eval months:
- VC@100 improved in 8/12 months -- sign test p-value ~0.19 (not significant at 0.05)
- The mean improvement of +0.0038 is small relative to the std of 0.065
- **Cohen's d ~ 0.06** -- a negligible effect size
- The improvement is real but small; it's plausible this is noise, though the consistency across multiple metrics (all improving) reduces that probability

## Recommendations for Next Iteration

1. **Tier-VC@100 is 0.0004 below floor** -- this is the ONLY blocking gate. The next iteration should target VC@100 specifically:
   - Consider **log-transformed features** (`log1p_hist_da`, `log1p_expected_overload`) -- these compress long-tailed price distributions and may help the model discriminate at the high end
   - Consider `prob_range_high = prob_exceed_100 - prob_exceed_110` which captures the density of flow in the critical 100-110% band
   - Additional interaction features using `recent_hist_da` (the #1 importance feature) with other physical features

2. **Tier-Recall@1 is a structural problem** that FE cannot fix. This needs class weight tuning (increase tier 1 weight from 5 to 8-10) or a dedicated tier 1 model. Flag this for a future non-FE batch.

3. **Macro-F1 failing** is also structural -- driven by the tier 1 recall collapse. Same root cause.

4. **Verify interaction feature population**: Check whether `expected_overload`, `prob_exceed_110`, `tail_concentration` are actually populated in the data loader output, or whether the WARNING about missing tier features means these interactions are zero-filled. If zero-filled, the 5.4% improvement is coming from random noise in the zero-fill pattern, not from actual signal.

5. **Pruning dead computed features**: The 5 dead interaction features in `compute_interaction_features()` should be removed to keep the code clean, but this is cosmetic.

6. **Value-QWK barely passing** (0.3918 vs floor 0.3914) -- monitor closely. Any change that degrades value-weighted ordinal consistency could flip this gate.
