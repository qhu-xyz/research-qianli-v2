From the available signals (low recall@1 + flat QWK), the model likely still under-identifies tier 1 and relies on broader ranking rather than improved class separation for rare high-value tiers.

### Recommendations
1. Align gate implementation to the documented contract: make NDCG and QWK hard gates, restore Tier-Accuracy and Adjacent-Accuracy outputs, and remove contract drift between `evaluate.py`, `compare.py`, and `gates.json`.
2. Add a leakage guard test around loader boundaries (assert train window excludes target month rows for each `period_type`, especially `f0`).
3. In the next non-FE batch, prioritize Tier-Recall@1 recovery via class-weight/hyperparameter tuning; FE-only changes did not move this needle.
4. Add month-level confusion matrix export and per-tier precision/recall to diagnose adjacent vs distant errors directly.

### Gate Calibration
- Current `noise_tolerance = 0.02` is too loose for low-scale metrics. Example: Tier-Recall@1 bottom-2 champion is 0.0030, so a 0.02 tolerance effectively guarantees Layer-3 pass even under major degradation.
- Use metric-scaled tolerance (e.g., `min(0.01, 0.5 * champion_std)` or a relative fraction of champion bottom-2) instead of one global absolute 0.02.
- Keep Tier-VC@100 floor unchanged at 0.075 for now; v0005 is still below by 0.00046 and should not be promoted on the money metric.
